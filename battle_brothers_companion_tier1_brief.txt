BATTLE BROTHERS RECRUIT POTENTIAL CALCULATOR — TIER 1 BUILD BRIEF
(Prepared for a future Claude Code session)

## Goal
A tool that evaluates a recruit at the HIRING SCREEN — not live in-combat stats,
not memory reading, not save-file parsing. Input the background, current stat
rolls, and star (talent) count, and get back: projected level-11 potential per
stat, an overall quality rating, and a recommended build archetype (or a clear
"not worth it" flag if nothing fits).

## Core mechanic (verified against Battle Brothers Wiki + community sources)
- Each background has its own min/max range for every starting attribute
  (documented on the wiki's "Character Backgrounds/Attribute Ranges" page,
  extracted from game files).
- At hire, a recruit rolls 0-3 stars ("talents") on three random attributes.
  Stars increase how much that stat can grow per level-up.
- Community-verified projection formula for level-11 potential (level 11 is
  the soft cap — stat growth and perks stop there):

    projected_stat = current_stat + (10 x avg_growth_per_level_for_that_stat)
                      + (stars_on_that_stat x 5)

  Each star is worth roughly +5 points by level 11. Per-attribute average
  growth rates are documented on the wiki's "Talents" page.

## Feature 1 — Potential calculator
Input per recruit:
- Background (dropdown, populated from data source #1)
- Current stat rolls: HP, Fatigue, Resolve, Initiative, Melee Skill,
  Ranged Skill, Melee Defense, Ranged Defense
- Which 3 attributes have stars, and how many (0-3 each)

Output:
- Projected level-11 value per stat (using the formula above)
- A simple per-stat rating relative to the background's known max range
  (e.g. how close to that background's ceiling this roll already is)

## Feature 2 — Build recommendation
Compare projected level-11 stats against known build archetype thresholds
(reference table below, paraphrased from the community guide above — these
are LEVEL-11 targets, so they compare directly against the calculator's
projected output, no conversion needed).

Reference thresholds (melee atk = matk, ranged atk = ratk, all values are
level-11 targets):

| Archetype                          | Key thresholds                                              |
|-------------------------------------|---------------------------------------------------------------|
| Shield User (frontline tank)        | matk 85+, mdef 15+, fat 120-130+, hp 70-80+, resolve 45       |
| Two-Handed Battle Forged Frontline  | matk 85+, mdef 30+, fat 130+, hp 70+, resolve 45-55            |
| One-Handed Battle Forged Frontline  | matk 85+, mdef 40+, fat 130+, hp 70+, resolve 45-55, rdef 10+  |
| One-Handed Nimble Dodge Frontline   | matk 85+, initiative 150+, mdef 25+, hp 70+, fat 100, resolve 40 |
| Sergeant                            | resolve 90+, fat 120+, hp 80+, mdef/rdef 10+ (matk 85+ nice-to-have) |
| Polearm Backline (Battle Forged)    | matk 85+, hp 80+, mdef 15+, fat 110-125+, resolve 40+, rdef 10+ |
| Polearm Backline (Nimble)           | matk 85+, initiative 150+, hp 80+, fat 110+, resolve 40+, rdef 10+ |
| Archer                              | ratk 85+, initiative 150+ (for Overwhelm builds), hp 80+, rdef 15+, fat 100+ |
| Crossbow man                        | ratk 85+, hp 80+, rdef 15+, fat 100+                            |
| Handgonne man                       | ratk 85+, hp 80+, rdef 15+, fat 100+ (initiative 150+ / resolve 60+ only for specific perk builds) |
| Thrower                             | ratk 85 (lower bar OK), hp 80+, rdef 15+, fat 100+              |

Logic:
- If projected stats clear one or more archetype thresholds with margin ->
  recommend that archetype (or archetypes, ranked by best fit)
- If projected stats fall short of every archetype threshold by a meaningful
  margin -> flag as "not worth investing further perks/gear into" (still
  usable as a cheap filler/backup, just don't sink resources into them)
- Borderline cases (close to one threshold) -> show as a marginal fit with
  the specific stat(s) that are the limiting factor

## Suggested tech stack (unchanged from earlier scope)
- Python
- PyQt5 or PySide6 for the overlay window (global hotkey to summon it,
  manual stat entry fields, always-on-top + transparent)
- `keyboard` or `pynput` for the global hotkey
- Simple dict/table lookups for backgrounds + growth rates + archetype
  thresholds — no database needed for a tool this size

## Explicitly out of scope for now
- No live memory reading of the running game (that's a possible future
  stretch goal, not part of this build — see prior notes on Cheat Engine
  community tables if that's ever revisited)
- No save-file parsing
- This is purely: hire-screen inputs -> projected potential -> build fit /
  cut recommendation
